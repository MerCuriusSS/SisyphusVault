## 第一部分：项目结构拆解（老兵视角）

该项目采用多模块 Maven 架构，你重点需要关注的是以下三个核心目录：

- **`ruoyi-common` (核心资产库)**：包含所有工具类、切面（AOP）、注解定义。面试中关于“你如何封装通用组件”的话题全在这里。
    
- **`ruoyi-framework` (核心配置层)**：集成了 Redis、安全框架、多租户拦截器。这里是体现你“架构理解”的核心。
    
- **`ruoyi-extend` (扩展功能)**：包含文件上传、邮件等。

## 第二部分：7 天冲刺大纲

#### 第 1-2 天：【基石与血脉】—— 架构体系与权限模型

- **目标：** 理解 Spring Boot 3.x 启动流程，吃透 Sa-Token 权限体系。
    
- **关键里程碑：** 能够独立描述一个请求从前端点击到后端 `Controller` 经过了哪些拦截器。
    
- **步骤：**
    
    1. **启动与观察：** 跑通项目，观察控制台 Log，看项目启动时加载了哪些核心 Bean。
        
    2. **安全框架：** 对比 Spring Security，研究 Sa-Token 路由拦截方案（查找 `SaServletFilter`）。
        
    3. **多终端适配：** 看项目如何通过一个 `LoginHelper` 处理不同端（PC/App）的登录状态隔离。
        

#### 第 3 天：【灵魂注入】—— AOP 与统一异常/结果处理

- **目标：** 找回 Java 核心开发的感觉，掌握规范的工程化处理。
    
- **关键里程碑：** 彻底理解 `GlobalExceptionHandler` 及其对不同业务异常的捕获逻辑。
    
- **步骤：**
    
    1. **日志切面：** 拆解 `LogAspect`，看它如何通过注解记录操作日志。
        
    2. **优雅返回：** 学习 `R<T>` 泛型包装类的设计，思考这在团队协作中的价值。
        

#### 第 4 天：【高级进阶】—— 多租户与 ORM 黑科技

- **目标：** 这是 5 年经验的“提分项”，掌握数据隔离的底层逻辑。
    
- **关键里程碑：** 能够解释 Mybatis-Plus 是如何通过拦截器在 SQL 运行前动态插入 `tenant_id` 的。
    
- **步骤：**
    
    1. **租户过滤：** 深入 `PlusDataPermissionHandler` 源码。
        
    2. **对象转换：** 学习 `MapStruct`（找一个接口看它如何替代 `BeanUtils`），准备好回答关于对象拷贝性能优化的面试题。
        

#### 第 5-6 天：【性能加持】—— Redis 深度实战与并发控制

- **目标：** 应对高并发面试题。
    
- **关键里程碑：** 掌握分布式锁（Redisson）和防重提交（幂等性）的实战写法。
    
- **步骤：**
    
    1. **分布式锁：** 寻找项目中的 `@Lock4j` 或 Redisson 注入点，理解 Watchdog 机制。
        
    2. **防重逻辑：** 观察 `RepeatSubmitAspect`，分析它如何利用 Redis 实现 Token 桶或简单计数限流。
        
    3. **缓存一致性：** 看项目在更新数据时，是如何同步更新 Redis 缓存的。
        

#### 第 7 天：【简历转化】—— 亮点总结与话术封装

- **目标：** 把这 6 天的积累转化为简历上的文字。
    
- **关键里程碑：** 产出 3 段可以直接写在简历上的“项目成就”。
    
- **步骤：**
    
    1. **话术提炼：** “基于 Spring Boot 3.2 重构了企业级多租户后台，引入 Sa-Token 实现了轻量化 RBAC 模型...”
        
    2. **痛点攻克：** 总结“在 5 年开发经验基础上，如何快速通过 AOP 解决系统幂等性与审计日志问题”。

### 第三部分：给你的避坑指南

1. **不要纠结前端：** Vue 代码一眼都不要看，你的目标是 Java 后端高级岗位。
    
2. **不要死记硬背：** 5 年经验面试官看重的是“决策权衡”。比如：_“为什么要用 Redisson 而不是自己写 Lua 脚本？”_ 你要从开发效率、可靠性、社区活跃度来回答。
    
3. **利用好“注释”：** RuoYi-Vue-Plus 的代码注释非常详细，Debug 时多读作者留下的中文注释。
